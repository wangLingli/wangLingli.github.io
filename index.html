LESS 原理及使用方式

Less 是一门 CSS 预处理语言，它扩展了 CSS 语言，增加了变量、Mixin、函数等特性，使 CSS 更易维护和扩展。

本质上，LESS 包含一套自定义的语法及一个解析器，用户根据这些语法定义自己的样式规则，这些规则最终会通过解析器，编译生成对应的 CSS 文件。LESS 并没有裁剪 CSS 原有的特性，更不是用来取代 CSS 的，而是在现有 CSS 语法的基础上，为 CSS 加入程序式语言的特性。

less与sass的区别

基本语法

sass支持不包含花括号与分号的编写风格，仅通过缩进表示嵌套,less不支持。

变量

LESS使用@，而Sass使用$。

条件语句和控制

这是一个很强大的功能，也是LESS不支持的功能。使用Sass，你可以使用if{}else{}这样的条件语句，以及for{}循环语句

less 使用关键字“when”实现条件语句

循环语句

less 通过when模拟循环功能

sass 使用@for关键字，配合“from”和“through”

作用域

less有局部变量和全局变量的概念，查找变量的顺序是先在局部定义中找，如果找不到，则查找上级定义，直至全局

sass无全局变量的概念，后定义的同名变量会完全覆盖先定义的变量

scss-作用域
$color: #00c; /* 蓝色 */

#header {

  $color: #c00; /* red */
  border: 1px solid $color; /* 红色边框 */
}

#footer {
  border: 1px solid $color; /* 蓝色边框 */
}

Sass-作用域编译后

#header{border:1px solid #c00}
#footer{border:1px solid #c00}

输出设置

Less没有输出设置，Sass提供4中输出选项：nested, compact, compressed 和 expanded。

输出样式的风格可以有四种选择，默认为nested

  • nested：嵌套缩进的css代码
  • expanded：展开的多行css代码
  • compact：简洁格式的css代码
  • compressed：压缩后的css代码

混合（Mixins）

less 可以无需特别声明，直接调用某一class或id选择器名即可重用该选择器内属性

sass 定义混合需要以“@mixin”开头。引用混合需要以“@include” 开头

let 和 const

let 的作用和 var 一样，都是用来声明变量，但是 let 只在代码块中有效。

let 不允许在相同作用域内，重复声明同一个变量。

const 声明一个只读的常量。一旦声明，常量的值就不能改变,也是只在代码块中有效。

箭头函数

箭头函数的语法很简单：标识符=>表达式。你无需输入 function 和 return，一些小括号、大括号以及分号也可以省略。

Promise

Promise是异步编程的一种解决方案，它有三种状态，分别是pending-进行中、resolved-已完成、rejected-已失败,当Promise的状态又pending转变为resolved或rejected时，会执行相应的方法，并且状态一旦改变，就无法再次改变状态。

then()方法传递的两个函数:

  • 第一个函数是 resolve() 的回调
  • 第二个函数是 reject() 的回调

class 类

  • ES6 用class定义了类，constructor 是构造方法，而 this 关键字则代表实例对象。

  • constructor 内定义的方法和属性是实例对象自己的，而 constructor 外定义的方法和属性则是所有实例对象可以共享的。

  • Class之间可以通过extends关键字实现继承,Cat类通过extends关键字，继承了Animal类的属性和方法。

  • super 关键字，它指代父类的实例。子类必须在constructor方法中调用super方法。

location.hash

Veröffentlicht am 2017-07-18

最近看了一个用swiper.js做的活动页项目，当滑到第六页点击“了解详情”按钮，可以使用指定相应的url跳转到详情页，那么当点击详情页中“返回”按钮，怎么返回到swiper第六页的活动页面呢？



window.location.href

location是javascript里边管理地址栏的内置对象,window.location.href可以获取到完整的url，比如window.location.href=”www.babytree.com”表示的是重新定向，页面跳转到新的页面。

window.location.hash

“#”代表网页中的一个位置,其右面的字符，就是该位置的标识符。比如，

http://www.babytree.com#tool

就代表网页的tool位置。浏览器读取这个URL后，会自动将tool位置滚动至可视区域。

为网页位置指定标识符，有两个方法。一是使用锚点，比如<a name="tool"></a>，二是使用id属性，比如<div id="tool" >。

获取的是页面的锚链接，相比如href,通过window.location.hash并不会跳转到新的链接，只会在当前链接里面改变锚链。例如<a href="www.babytree.com#tool">通过window.location.hash得不到完整的链接（URL），仅得到 #tool.

window.location.hash这个属性可读可写。读取时，可以用来判断网页状态是否改变；写入时，则会在不重载网页的前提下，创造一条访问历史记录。

每一次改变#后的部分，都会在浏览器的访问历史中增加一个记录，使用”后退”按钮，就可以回到上一个位置。这对于ajax应用程序特别有用，可以用不同的#值，表示不同的访问状态，然后向用户给出可以访问某个状态的链接。

值得注意的是，上述规则对IE 6和IE 7不成立，它们不会因为#的改变而增加历史记录。

实现步骤

给返回按钮设置带有标签值的a链接，<a href="/sakykids#referno5"></a>

获取页面锚链接：

var hash = window.location.hash;  // #referno5

获取swiper第几页序号：

var referno = /referno/.test(hash)?hash.split('referno')[1]:0; //5

防止花括号闪烁

打开页面有时候会出现 {{msg}} 这种表达式的情况，这是因为 JavaScript 会等DOM加载完成之后，才会去操作相应的DOM元素进行数据渲染。

使用v-text

在vue中可以使用 v-text 指令渲染数据，这样可以避免 {{ }} 的产生，如果里面包含需要解析的 html 元素，则使用 v-html即可。

使用v-cloak

v-cloak一般用在大段落中包含大量 {{ }} 的div上，这个指令保持在元素上直到关联实例结束编译。和 CSS 规则如 [v-cloak] { display: none } 一起用时，这个指令可以隐藏未编译的 Mustache 标签直到实例准备完毕。

slot插槽

slot是一个很有用的特性，可以用来向组件内部插入一些内容。slot就是“插槽”的意思，相当先定义好位置，由用户来决定插入的内容，这样就不会直接覆盖原有的内容。

默认情况下页面只会显示“我是子组件”，而ul里的内容不显示：

匿名slot

单个slot使用最简单，也是最常用的，父组件在使用的这个组件的时候，想在内部自定义一些初始化数据，这时候就可以用slot实现。

当child组件内有多个标签也会一起插入到slot的位置。

具名slot

  • 将放在子组件里的不同html标签放在不同的位置
  • 父组件在要分发的标签里添加 slot=”name名” 属性
  • 子组件在对应分发的位置的slot标签里，添加name=”name名” 属性，
  • 然后就会将对应的标签放在对应的位置了。

简介

MVVM 是Model-View-ViewModel 的缩写，它是一种基于前端开发的架构模式，其核心是提供对View 和 ViewModel 的双向数据绑定，这使得ViewModel 的状态改变可以自动传递给 View，即所谓的数据双向绑定。

Vue.js 是一个提供了 MVVM 风格的双向数据绑定的 Javascript 库，专注于View 层。它的核心是 MVVM 中的 VM，也就是 ViewModel。 ViewModel负责连接 View 和 Model，保证视图和数据的一致性，这种轻量级的架构让前端开发更加高效、便捷。 （官网地址：https://cn.vuejs.org/）

常用指令

  • v-for： 循环指令，基于一个数组渲染一个列表，与JavaScript遍历类似，需要以 item in items 形式的特殊语法。

获取index索引：

  • v-show：也是条件渲染指令，和 v-if 指令不同的是，使用 v-show 指令的元素始终会被渲染到HTML，v-show 只是简单的把元素的 csss属性改为 “display:none”。

  • v-html：与v-show不同的是会将数据中的 html 解析出来。

  • v-if：根据表达式的值的真假条件渲染元素。

  • v-else：条件渲染，v-else 元素必须紧跟在 v-if 或者 v-else-if 元素的后面——否则它将不会被识别。

  • v-bind：可以在其名称后面带一个参数，中间放一个冒号隔开，这个参数通常是HTML元素的特性（attribute），例如：v-bind:class

组件生命周期

  • beforeCreate ：组件实例刚刚被创建,属性都没有绑定。
  • created ：实例已经创建完成，属性已经绑定。
  • beforeMount ：模板编译之前。
  • mounted ：模板编译之后。
  • beforeUpdate ：组件更新之前。
  • updated ：组件更新完毕。
  • beforeDestroy：组件销毁前。
  • destroyed ：组件销毁后。
